## Scope and method

The authoritative upstream comparison touched **102 coding-agent files** (**3,514 additions and 627 deletions across 47 coding-agent-touching commits**). The earlier 97-file subtotal covered the named source, test, docs, examples, README, and changelog surfaces but omitted five package/config/generated files: `package.json`, `npm-shrinkwrap.json`, `install-lock/package.json`, `install-lock/package-lock.json`, and `vitest.config.ts`. Atomic did not wholesale-copy those files. Every upstream change was compared with Atomic's split/refactored fork, then classified as an issue-required fix, dependency-inherited behavior, an existing Atomic equivalent, an intentional non-applicable divergence, or additive upstream feature work outside #1703's fix contract.

The implementation ports only applicable behavior and preserves Atomic branding, `@bastani/*` package names, the `atomic` CLI, `.atomic` paths, bundled first-party extensions, virtual-module compatibility aliases, versionless `0.0.0` manifests, Verbatim Compaction, hashline editing, the split standalone launcher/archive layout, lock-free auth reads, and Atomic-specific Copilot/Gemini model, context-window, retry, sanitization, and streaming repairs.

## Other audited v0.80.4-v0.80.6 surfaces

Upstream changed CLI/config, agent session, auth, compaction, extension loading/types, model registry/resolution, resource loading, session management, settings, bash/edit tools, SDK exports, interactive components/theme, RPC/print modes, and clipboard utilities. Issue-applicable fixes are accounted for above. Atomic additionally ports v0.80.6 request-wide cost-tier schemas and propagation through custom models, partial overrides, and extension providers. Atomic's dual-file compatibility layer now composes legacy and primary override maps by provider/model ID with complete primary-entry precedence and provenance-aware headers. The dependency type expansion also required Atomic to carry `max` thinking through CLI/settings/session/TUI, Cursor mapping, workflows, subagents, and the preset example. Upstream tests were used as behavioral references, then adapted to Atomic's split modules and deliberate divergences. New or extended Atomic coverage exists for dependency pricing/catalog behavior, custom/override/extension request-wide cost tiers, normal-startup and two/three-layer primary/legacy override composition, the full-control SDK example's catalog resolution, stale usage, null and omitted content ingress plus derived Verbatim Compaction paths, restricted dynamic restoration, Windows traversal, auth persistence, timeout validation, label reset, session-id warning, fork/custom-message ordering, real isolated all-target clipboard staging/copy, missing-wrapper `--skip-deps`, external-cwd caller-relative TMPDIR behavior, clipboard fallback/layout, `max` reasoning, Cursor mapping, workflows, and subagents. The upstream range changed `custom-provider.md`, `extensions.md`, `models.md`, `packages.md`, `rpc.md`, `sdk.md`, `session-format.md`, `settings.md`, `themes.md`, `tui.md`, and `usage.md`; extension/SDK/provider examples; the coding-agent README; and released changelog sections. The preset example accepts `max`. No entry-renderer example was added because that API is absent. Only Atomic `Unreleased` changelog sections were changed, plus the current July 10 docs-site update; upstream released changelog sections were not copied or edited.
